Waar wordt de event listener gekoppeld aan de slider?

Bij sliders[0].addEventListener("change", update);

Waarom moeten we die op twee soorten events koppelen?

De change event reageert wanneer de muis losgelaten word (bij Chrome), het update event dient voor wanneer de slider verplaatst aan het worden is.

In de CSS file wordt nergens een rode kleur opgegeven, waar wordt dan wel de rode kleur van het blokje ingesteld?

In de script file wordt het element met class colordemo gecalled en een rode achtergrondkleur toegewezen

Waarom schrijven we telkens sliders[0] en colorDemos[0] en niet gewoon sliders of colorDemos?

Bij het callen van getelementsbyclassname maak je een HTMLcollection van alle elementen die de klasse "colorDemos"/"sliders" hebben. Daarom moet je gaan aangeven op welke positie in de collection het element zich bevind. Hier omdat de collection maar 1 element bevat [0]. Het 1ste element in de collection.

Waar vind je de CSS-properties die programmatorisch werden ingesteld, zoals bv. de rode achtergrondkleur van het blokje?

Door te kijken in de devtools console. Als een element een inline style blok heeft dan is deze programmatorisch ingesteld geweest.
